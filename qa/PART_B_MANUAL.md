# QA Part B — 사장님용 수동 검증 가이드

Part B는 Chrome 확장 UI 동작 확인이라 사장님 PC에서 직접 수행이 필요합니다(총 10~15분).
`03_qa_plan.md` Part B 10항목을 순서대로 진행.

## 사전 준비 (2분)
1. Supabase 대시보드 SQL Editor에서 `db/schema.sql` 실행 완료 (미완 시 Part A도 실패)
2. 터미널: `cd C:\dev\dalgeumi-desk && npm run build` 실행 → `dist/` 생성 확인

## B1. 확장 로드
1. Chrome 주소창 → `chrome://extensions`
2. 우측 상단 **개발자 모드** ON
3. 좌측 **압축해제된 확장 프로그램 로드** 클릭 → `C:\dev\dalgeumi-desk\dist` 선택
4. ✅ 에러 없이 로드되고 카드에 "달그미데스크" 표시
5. 새 탭 열기 → 로그인 화면이 떠야 함

## B2. Magic Link OTP 로그인
1. 새 탭에서 이메일 입력 (회사 계정 권장) → "로그인 코드 받기"
2. 이메일함에서 Supabase가 보낸 메일 확인 (첫 사용 시 스팸함도 체크)
3. 6자리 코드 또는 메일 본문에서 숫자 코드를 확장에 입력 → "로그인"
4. ✅ Dashboard 렌더 + 이메일이 오른쪽 상단에 표시

## B3. 섹션 CRUD
1. "+ 섹션 추가" → "테스트섹션" 입력
2. 섹션 제목 클릭 → "수정됨"으로 이름 변경 프롬프트
3. 섹션 × 버튼으로 삭제
4. ✅ 3개 동작 모두 3초 이내 UI 반영

## B4. 북마크 CRUD
1. 섹션 하나 남긴 채, "+ 북마크 추가" → 제목 "Google" / URL `https://google.com`
2. 추가된 북마크 클릭 → 새 탭에서 google.com 열림 확인
3. × 버튼으로 삭제
4. ✅ favicon 표시(또는 G 이니셜 fallback), 링크 정상 작동

## B5. Realtime 멀티 창 동기화
1. 같은 계정으로 새 탭 2개 열기 (창 A, 창 B)
2. 창 A에서 북마크 추가
3. 창 B를 보자
4. ✅ **새로고침 없이** 3초 이내 창 B에도 동일 북마크 표시

## B6. 오프라인 캐시
1. F12 → **Network** 탭 → Throttling 을 **Offline** 으로
2. 새 탭 열기
3. ✅ 상단에 주황색 **"오프라인"** 배지 표시
4. ✅ 기존 섹션·북마크 모두 정상 렌더 (빈 화면 X)
5. 📸 스크린샷: 이 상태를 `qa-screenshots/B6_offline_cache.png` 로 저장

## B7. 오프라인 중 쓰기
1. B6 상태에서 "+ 섹션 추가" 시도
2. ✅ 빨간 에러 배너 "오프라인 상태입니다…" 표시

## B8. 온라인 복귀
1. Network Throttling 을 **No throttling** 으로 해제
2. ✅ 배지 사라짐, 이후 쓰기 정상 동작

## B9. 재부팅 후 데이터 유지
1. PC 재부팅
2. Chrome 열어 새 탭
3. ✅ 로그인 상태 유지되고 이전 데이터 그대로 표시

## B10. RLS 교차 차단
1. 우측 상단 "로그아웃" → 다른 이메일로 로그인
2. ✅ 새 계정 Dashboard 는 **완전히 비어있음** (이전 계정 데이터 보이면 FAIL)
